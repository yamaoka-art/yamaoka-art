# Raspberry Pi 4 自動化システム構成

## 概要

Garmin Connect・Arboleaf体重計からのデータを自動取得し、Markdownで記録・GitHub同期する完全自動化システム。

---

## 実行環境

- **OS**: Raspberry Pi OS 64-bit (aarch64)
- **Python**: 3.9.2
- **Node.js**: v24.16.0
- **ネットワーク**: 192.168.2.115 (Wi-Fi, DHCP)

---

## インストール済みライブラリ

- `anthropic` (Python Claude API)
- `garminconnect` (Garmin接続)
- `dropbox` (Dropbox API)

---

## ファイル構成
/root/

├── gym_tracker.py              # ジム記録自動化スクリプト

├── client_secret_health.json   # Google Health API認証情報

├── .bashrc                      # 環境変数設定

└── ObsidianVault/              # Git同期フォルダ

├── ジム記録_2026-06.md

├── 体重記録_2026-06.md

└── システム構成_自動化.md

## crontab設定（実行済み）

```bash
0 21 * * * /usr/bin/python3 /root/gym_tracker.py >> /root/gym_tracker.log 2>&1
0 22 * * * /usr/bin/python3 /root/weight_tracker_dropbox.py >> /root/weight_tracker.log 2>&1
```

---

## スクリプト説明

### gym_tracker.py
- 毎日21時にGarmin Connectから筋トレ記録を取得
- ジム記録_2026-06.mdに保存・Git push

### weight_tracker_dropbox.py
- 毎日22時にDropboxからHealth Connectデータを取得
- SQLiteで体重データを抽出して体重記録_2026-06.mdに保存・Git push

## health_evaluation.py（2026-06-21 追加）

**目的**: ジム記録と体重記録からClaudeが一日の体の変化を評価し、Discord通知

**実行時刻**: 毎日09:00

**処理内容**:
1. ジム記録_2026-06.md と体重記録_2026-06.md を読み込み
2. Claude API で健康評価を生成
3. Discord Webhook経由でスマホに通知

**Discord設定**:
- サーバー: raspberry-pi-health
- チャンネル: health-notifications
- Webhook URL: 環境変数 DISCORD_WEBHOOK に保存
